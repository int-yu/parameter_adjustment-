# DAPLink 串口调试台

通过浏览器 Web Serial 连接 DAPLink 虚拟串口的通用数据包调试工具。页面在本地浏览器内运行，不包含后端服务，不执行 SWD 烧录，也不会自动重连或补发离线期间的控制值。

发布地址：<https://int-yu.github.io/parameter_adjustment-/>

## 使用条件

- 桌面版 Chrome 或 Edge。
- 页面必须运行在 HTTPS 或 `localhost` 安全上下文中。
- DAPLink UART 与单片机 UART TX/RX/GND 正确连接并共地。
- 串口参数在“设置”页配置，默认是 `115200 8N1`、无流控。

## 功能页

### 协议终端

- 根据“数据包配置”中的 RX 结构解析收到的数据帧。
- 未知消息 ID 只要帧头、长度、CRC 和帧尾有效，仍会保留原始 payload。
- 支持按消息过滤、暂停当前视图、清空历史、导出 CSV。
- 支持结构化 TX 帧、原始 HEX、文本发送。
- 文本发送和原始日志解码支持 UTF-8、GBK、ASCII；结构化 `fixed-string` 按协议固定为 UTF-8。

### 展示终端

- 左侧是接收时间为 X 轴的曲线图。
- 右侧列出 RX 数据包中的所有整形和浮点变量。
- 每个变量可单独选择是否绘图，并配置颜色和缩放倍率。
- 曲线窗口长度在“设置”页配置。

### 专业调试

- 用户可创建按钮、开关、滑动条、双轴摇杆、数值输入槽。
- 控件绑定 TX 数据包中的特定字段，按钮/开关绑定 `bool`，滑动条/数值槽绑定整形或浮点型，摇杆绑定同一 TX 消息里的两个数值轴。
- 控件变化时发送所属 TX 消息的完整当前帧。
- 滑动条和摇杆拖动期间按 50 ms 合并发送，释放时发送最终值。
- 画布左下角提供移动模式和角度模式；控件位置和角度保存在 profile 中。

### 数据包配置

- 左右分区编辑发送数据包结构和接收数据包结构。
- 支持新增、复制、删除、上移、下移消息和字段。
- 字段可配置变量名、key、类型、固定长度、单位、默认值。
- 修改先进入草稿，点击“应用配置”后才写入当前 profile。
- 应用时会校验消息 ID 范围、重复 ID、字段 key、固定长度和 512 字节 payload 上限。
- 删除已绑定的数据包或字段时，会提示并解除相关曲线/控件绑定，控件布局仍保留。

### 设置

- 调整波特率、数据位、停止位、校验、流控。
- 调整文本编码、文本发送行尾、帧历史、日志历史、曲线时间窗口。
- 导入、导出或重置 profile v2。旧 v1 PID 配置不会自动迁移。

## 二进制帧协议 v1

所有多字节整数和浮点数均为小端序。每个消息 ID 对应一组 payload 字段结构。

| 字节偏移 | 长度 | 字段 | 说明 |
|---:|---:|---|---|
| 0 | 1 | HEAD0 | 固定 `0xAA` |
| 1 | 1 | HEAD1 | 固定 `0x55` |
| 2 | 1 | VERSION | 固定 `0x01` |
| 3 | 1 | MESSAGE_ID | 消息 ID |
| 4 | 2 | SEQUENCE | `uint16`，小端，循环递增 |
| 6 | 2 | PAYLOAD_LENGTH | `uint16`，小端，最大 512 |
| 8 | N | PAYLOAD | 按字段顺序连续排列，无填充 |
| 8+N | 2 | CRC16 | 小端 |
| 10+N | 1 | TAIL0 | 固定 `0x0D` |
| 11+N | 1 | TAIL1 | 固定 `0x0A` |

CRC 使用 `CRC-16/CCITT-FALSE`：

- Polynomial: `0x1021`
- Init: `0xFFFF`
- RefIn/RefOut: `false`
- XorOut: `0x0000`
- 校验范围：从 `VERSION` 到 payload 最后一个字节。
- 标准字符串 `123456789` 的结果为 `0x29B1`。

消息 ID 范围：

| 范围 | 方向 |
|---|---|
| `0x01`-`0x7F` | 设备到网页 |
| `0x80`-`0xEF` | 网页到设备 |
| `0xF0`-`0xFF` | 保留 |

字段类型：

| 类型 | 字节数 | 编码 |
|---|---:|---|
| `bool` | 1 | 只能为 `0x00` 或 `0x01` |
| `u8` / `i8` | 1 | 无符号/有符号整数 |
| `u16` / `i16` | 2 | 小端整数 |
| `u32` / `i32` | 4 | 小端整数 |
| `f32` | 4 | IEEE-754 单精度，小端 |
| `f64` | 8 | IEEE-754 双精度，小端 |
| `fixed-string` | 配置长度 | UTF-8，NUL 补齐，超长时按完整 Unicode 字符截断 |
| `fixed-bytes` | 配置长度 | 原始固定长度字节，不足补 0 |

协议 v1 不支持结构体自动对齐、可变长字符串、嵌套结构或可变长数组。单片机端应按字段顺序逐字节打包，避免直接发送带编译器填充的 C 结构体。

```powershell
npm install

npm run build

```

生产构建输出到 `dist/`。Vite base 固定为 `/parameter_adjustment-/`，适配 GitHub Pages。

## 浏览器与安全限制

- Web Serial 需要用户点击连接后手动授权 COM 口。
- 浏览器不能静默获取串口权限。
- 网页不会自动重连、自动发送历史控件值或在断线后补发。
- 实机首次测试建议先断开电机动力或架空车轮，确认消息 ID、字段顺序、CRC、方向和输出范围。
